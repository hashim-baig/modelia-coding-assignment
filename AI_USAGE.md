# AI Usage

Describes how AI (and human effort) contributed to the development of this project.

---

## 🤖 AI Contributions

- **Component scaffolding**
  Generated initial skeletons for: `Upload`, `PromptInput`, `StyleSelector`, `Preview`, `Spinner`, `ErrorBoundary`, `GenerateButton`, `History`.
- **Utilities**
  Wrote initial mocks and helpers for downscaling, history, and API (with error and retry logic).
- **Testing**
  Generated example unit test templates for components and utilities, and Cypress E2E sample flows.
- **Documentation**
  Drafted the first versions of `README.md`, `AI_USAGE.md`, and PR templates.
- **Scaffolding UI and best practices**
  Provided boilerplate (file structure, configs), accessibility notes, and suggestions for state/data flow.

---

## 👨‍💻 Human Contributions

- **UI/UX and Styling**
  Enhanced layouts (sidebar, toasts, chat-style prompt box).
  Major design decisions, responsiveness, consistency improvements.
- **Interaction refinement & feedback**
  Real-world usability improvements (toasters, spacing, error handling, abort/retry flows).
- **Refactoring and code review**
  Streamlined components, reorganized structure, DRY-ed code.
- **Accessibility & semantics**
  Added focus management, aria fields, improved keyboard navigation.

---

## 📝 Takeaways

AI greatly accelerated boilerplate and helped with common patterns and test templates, especially early on.
All product-level logic, UX, error handling, and architecture decisions were shaped by human review, iteration, and domain understanding.
